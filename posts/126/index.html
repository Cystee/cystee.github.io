记录

0002-05-06
#MAIN  #MAIN: untagged 

由于这个博客更新之后，首页不会再是乱糟糟的了，所以在本“记录”模块下，我可以肆无忌惮地抄那些看起来乱糟糟的代码。

上一篇文章看起来应该放入另一个“传播”模块，不过由于它是确实记录了这次更新，而且是更新后的第一篇文章，所以最合理的方式应该是放入本“记录”模块，就像现在这样，作为本“记录”模块的第一篇文章。

然而博主发现其实没什么好记录的了，所以往后可能本“记录”模块只会存在这一篇和上一篇两篇文章了。

为了体现它是用来记录的，就稍微记录一下这次博客程序的更新，即如何页面居中一个元素。

  1. 存在一个需要页面居中的元素 <div id="need-center"></div>
  2. 设置本元素的 CSS： 
    #need-center {
    position: absolute;
    top: 50%;
    transform: translateY(-50%);
}
    • position 设置为 absolute，绝对定位
    • 两个 50% 可以调整
  3. 采用这种定位方式，缩放页面也许会造成溢出，这时通过限制 max-height 或 max-width 以及添加 overflow 属性来控制元素的大小以及溢出时的处理方式

另外，通常使页脚居底，采用类似的方式：

CSS 存在一个元素.footer 
.footer {
    position: absolute;
    bottom: 0;
    width: 100%; /* 关键 */ 
}

好，本篇文章完了，以后这种“记录”大概率也不会有了。
